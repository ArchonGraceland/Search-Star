1. Overview

Search Star is an open-source standard for self-sovereign personal data profiles. An individual hosts their profile at a URI they control, served as JSON-LD. Platforms — dating apps, recruiters, investment networks, brands — query the individual's API endpoint and pay per query at a price the individual sets.

The standard covers three data domains: financial standing (expressed as age-cohort percentiles, not raw dollars), personal presence (the Presence Composite: Rizz, Vibe, Drip), and professional capability (skills, credentials, interests). Each field can be independently scoped to one of three access tiers.

Search Star inverts the data economy. Instead of platforms harvesting user data for free and selling it to advertisers, individuals own, host, price, and monetize their own data directly.

2. Identity & Registration

Each Search Star profile is assigned a permanent, sequential identifier in the format SS-XXXXXX (e.g., SS-000001). This number never changes and is globally unique.

Identity is anchored by a W3C Decentralized Identifier (DID) using the did:web method, resolving to a domain the profile owner controls. KYC verification is required and attested by a third-party provider (e.g., Stripe Identity). The DID, KYC attestation, and profile number together form the identity triple.

3. Profile Schema

Profiles are served as JSON-LD documents at the owner's endpoint. The schema context is https://schema.searchstar.org/v0.1. All fields support field-level access control via the accessTier property.

3.1 Financial (Age-Percentile Model)

All financial metrics are expressed as percentiles within the owner's age cohort (5-year bands). No raw dollar amounts ever leave the profile. This makes a 26-year-old with $100K in savings and a 50-year-old with $2M directly comparable — both might be 90th percentile for their cohort.

Percentile data is sourced from the Federal Reserve Survey of Consumer Finances and verified via zero-knowledge proofs (ZK-SNARKs). The proof attests that the owner's actual value meets or exceeds the claimed percentile threshold, without revealing the underlying number.

3.2 Presence Composite™

The Presence Composite is a three-dimensional score measuring interpersonal magnetism, aesthetic taste, and visual self-expression. It is deliberately brand-blind — the algorithm has no input for price tags, brand names, or logos.

Three dimensions

DimensionDefinitionFactors
RizzInterpersonal magnetism & charismaWit, warmth, presence, confidence
VibeEnergy, taste & the world you curateAesthetic sense, cultural range, consistency, environment
DripVisual expression & personal styleFit & silhouette, color sense, originality, context match

Scoring penalties & boosts

The Drip dimension includes explicit penalties and boosts to prevent the score from becoming a brand-spending leaderboard:

PenaltyVisible logos-0.15
PenaltyBrand homogeneity-0.10
PenaltyPrice-as-personality-0.20
BoostOriginality+0.20
BoostCoherence+0.15
BoostRange+0.10

Data sources

Presence scores are derived from: photo AI analysis (fit, color, composition), blinded peer reviews (reviewers cannot see others' scores), space photos (home and workspace aesthetic), taste signals (music, film, reading), anonymized conversation samples, and social content originality indexes.

3.3 Skills & Credentials

Skills use the W3C Verifiable Credentials standard. Each skill has a name, self-reported level (beginner through expert), a numeric score (0–100), and an optional verifiable credential with issuer DID and cryptographic proof.

3.4 Interests

Interests are organized into three domains:

Interests are part of the public tier by default — they serve as discovery signals that help platforms and people find relevant profiles.

4.2 Pricing Model

All pricing is per-query (or per-message for Marketing tier), set by the profile owner. There are no subscriptions, no flat rates, and no free tiers. Owners can set different prices per platform. Billing is handled via Stripe Connect with real-time settlement.

A message costs what it costs. There are no refunds on Marketing tier messages. The price is the filter — it protects the profile owner's attention and signals seriousness from the sender.

4.3 Marketing Tier Rules

The Marketing tier allows anyone to message a profile owner by paying their set price. Rules enforced by the protocol:

  • Maximum message length: 500 characters
  • Sender must verify identity before sending
  • Profile owner can permanently block any sender
  • No refunds — payment is for the opportunity to reach the owner, not a guarantee of response

Future: response likelihood score

A planned addition to the Marketing tier is a response likelihood score displayed to senders before payment. Based on the profile owner's historical response rate by message category (recruiter, brand, dating, investor), this score lets senders make an informed decision about whether the price is worth it for their specific use case.

5. Profile Numbering

Profiles are assigned sequential numbers in the format SS-XXXXXX, zero-padded to six digits. The number is permanent — it never changes, even if the owner changes their handle or domain. Numbering starts at SS-000001.
